Descrição detalhada

O coração do jogo, gerenciando o loop principal, estados e componentes.

A classe GameEngine é responsável por inicializar o Allegro, carregar recursos, controlar o fluxo entre as diferentes telas (menu, jogo, game over, ranking, configurações), processar eventos de entrada e gerenciar o loop de atualização e renderização do jogo.

void GameEngine::run ( )

Inicia e executa o loop principal do jogo.

O loop principal de execução do jogo.

Este método contém o ciclo de vida do jogo: processamento de eventos, atualização da lógica e renderização, até que o jogo seja encerrado.

Este método orquestra a inicialização do Allegro, o carregamento dos assets, e o ciclo de vida do jogo, que inclui:

  1. Processamento de eventos (entrada do usuário, timer, fechamento da janela).
  2. Atualização da lógica do jogo (chamando update).
  3. Renderização dos elementos na tela (chamando render apropriado). O loop continua até que a flag fecharJogo seja definida como true.
706 {
707 initializeAllegroAddons(); // Inicializa o Allegro e seus add-ons.
708 loadGameAssets(); // Carrega todos os recursos do jogo.
709
710 al_start_timer(timer); // Inicia o timer para controlar a taxa de quadros (FPS).
711 bool redraw = false; // Flag para indicar se a tela precisa ser redesenhada.
712
713 // Loop principal do jogo. Continua executando enquanto a flag 'fecharJogo' for falsa.
714 while (!fecharJogo) {
715 ALLEGRO_EVENT ev;
716 // Espera por um evento na fila, o que economiza CPU quando não há eventos.
717 al_wait_for_event(queue, &ev);
718
719 if (ev.type == ALLEGRO_EVENT_TIMER) {
720 // Se for um evento do timer, é hora de atualizar a lógica do jogo.
721 update(1.0 / 60.0); // Atualiza o jogo com base em um delta time de 60 FPS.
722 redraw = true; // Marca a flag para redesenhar a tela.
723 } else if (ev.type == ALLEGRO_EVENT_DISPLAY_CLOSE) {
724 // Se o usuário clicou no botão de fechar a janela.
725 fecharJogo = true; // Define a flag para sair do loop principal.
726 } else {
727 // Qualquer outro evento (teclado, mouse) é passado para o handler de input.
728 handleInput(ev);
729 }
730
731 // Só redesenha a tela se a flag 'redraw' estiver true e a fila de eventos estiver vazia.
732 // Isso evita redesenhar múltiplas vezes para o mesmo frame, otimizando a performance.
733 if (redraw && al_is_event_queue_empty(queue)) {
734 redraw = false; // Reseta a flag após o redesenho.
735 al_set_target_bitmap(renderTarget); // Redireciona o desenho para o bitmap temporário.
736 al_clear_to_color(al_map_rgb(0, 0, 0)); // Limpa o buffer temporário com preto.
737
738 // Chama a função de renderização correta baseado no estado atual do jogo.
739 switch (estadoAtual) {
740 case MENU:
741 renderMenu(1.0 / 60.0);
742 break;
743
744 case INICIANDO_JOGO: {
745 // Renderiza o cenário ou o menu se o cenário ainda não foi criado.
746 if (scenario) {
747 scenario->render();
748 } else {
749 renderMenu(1.0f / 60.0f);
750 }
751
752 // Desenha o efeito de fade-out/in da transição.
753 float alpha = 1.0f - (transitionBlurTimer / TRANSITION_BLUR_DURATION);
754 // Garante que o valor de alpha esteja entre 0.0 e 1.0.
755 if (alpha < 0.0f) alpha = 0.0f;
756 if (alpha > 1.0f) alpha = 1.0f;
757 al_draw_filled_rectangle(0, 0, screenWidth, screenHeight, al_map_rgba_f(0, 0, 0, alpha));
758 break;
759 }
760
761 case JOGANDO:
762 renderGame();
763 break;
764 case GAME_OVER:
765 renderGameOver();
766 break;
767 case RANKING:
768 renderRanking(1.0 / 60.0);
769 break;
770 case CONFIG_SCREEN:
771 renderConfigScreen(1.0 / 60.0);
772 break;
773 }
774
775 al_set_target_backbuffer(display); // Volta a renderizar para o display principal.
776 // Desenha o conteúdo do bitmap temporário na tela, aplicando escalonamento se necessário.
777 al_draw_scaled_bitmap(renderTarget, 0, 0,
778 al_get_bitmap_width(renderTarget),
779 al_get_bitmap_height(renderTarget),
780 0, 0, screenWidth, screenHeight, 0);
